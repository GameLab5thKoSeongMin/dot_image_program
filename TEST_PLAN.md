### Image Conversion Tests
- [x] Result image is exactly 32x32.
  - Result: Pass

- [x] Original image is divided into 32x32 tiles.
  - Result: Pass
  - Note: `calculateTileBounds` is called for each output pixel coordinate.

- [x] Median color is applied.
  - Result: Pass
  - Note: `calculateTileMedianColor` collects tile channel values and calls `calculateMedian`.

- [x] Transparent PNG is handled correctly.
  - Result: Pass
  - Note: Fully transparent and transparent-shape synthetic cases generate 32x32 output without errors, using transparent pixel rules.

- [x] Conversion uses tile median, not simple resize.
  - Result: Pass
  - Note: The output is written from tile medians into `ImageData`, not generated by drawing the original image into a 32x32 canvas.

- [x] Image smaller than 32x32 is handled without error.
  - Result: Pass
  - Note: The synthetic 17x17 image test passed; bounds are clamped to sample at least one pixel.

- [x] Large image generates a result.
  - Result: Pass
  - Note: A synthetic 1920x1080 image passed in both the app flow and the test page.

- [x] Very wide image is handled.
  - Result: Pass
  - Note: The synthetic 320x40 image test passed.

- [x] Very tall image is handled.
  - Result: Pass
  - Note: The synthetic 40x320 image test passed.
