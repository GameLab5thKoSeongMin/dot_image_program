# Core Goal

The goal of this program is to let the user input a PNG or JPG image, divide that input image into a 32-column × 32-row grid, calculate the representative color of each tile using the median value, and generate a final 32×32 pixel icon image.

This is important.

This program must not simply resize the image to `32×32`.

It must divide the original image into 32 columns and 32 rows, then compress each tile into one output pixel by calculating the median color value of the pixels inside that tile.

Do not finish the implementation using only a simple resize operation such as:

```js
ctx.drawImage(image, 0, 0, 32, 32);
```

That approach is just image resizing and does not satisfy the requirement.

The required approach is tile-based median color sampling.

## 3-2. 32×32 Tiling

Divide the input image into 32 columns and 32 rows.

This produces 1024 tiles.

Divide the original image width into 32 parts and the original image height into 32 parts.

Each tile corresponds to one pixel in the final 32×32 image.

Examples:

* Top-left tile of the original image → output pixel at `(0, 0)`
* Bottom-right tile of the original image → output pixel at `(31, 31)`

If tile boundaries are fractional, still make sure the whole image area is covered.

---

## 3-3. Tile Boundary Calculation Rules

The 32×32 conversion algorithm should calculate tile boundaries like this:

```js
const xStart = Math.floor(tileX * imageWidth / OUTPUT_SIZE);
const xEnd = Math.floor((tileX + 1) * imageWidth / OUTPUT_SIZE);
const yStart = Math.floor(tileY * imageHeight / OUTPUT_SIZE);
const yEnd = Math.floor((tileY + 1) * imageHeight / OUTPUT_SIZE);
```

If the original image is very small or tile boundaries overlap, causing an empty tile, adjust the bounds so that at least one pixel is sampled.

Example:

```js
const safeXEnd = Math.max(xEnd, xStart + 1);
const safeYEnd = Math.max(yEnd, yStart + 1);
```

Also clamp the bounds so they do not exceed the image area.

```js
const clampedXStart = Math.max(0, Math.min(xStart, imageWidth - 1));
const clampedYStart = Math.max(0, Math.min(yStart, imageHeight - 1));
const clampedXEnd = Math.max(clampedXStart + 1, Math.min(safeXEnd, imageWidth));
const clampedYEnd = Math.max(clampedYStart + 1, Math.min(safeYEnd, imageHeight));
```

This rule must allow stable handling of:

* 100×100 images
* 1920×1080 images
* 17×17 images
* Images smaller than 32×32
* Images with extreme aspect ratios

---

## 3-4. Tile Representative Color Calculation

Collect the pixels inside each tile.

Calculate the median value for each channel.

Basic calculation:

* R channel median
* G channel median
* B channel median
* A channel median

Final pixel color:

```txt
(medianR, medianG, medianB, medianA)
```

For transparent PNG handling, apply these rules:

1. If most pixels in a tile are transparent, the output pixel must also be transparent.
2. Pixels with very low alpha may be excluded from color median calculation.
3. The transparency threshold must be managed as a constant in `constants.js`.
4. Default: pixels with `alpha < 16` are considered transparent.
5. If the opaque pixel ratio inside a tile is too low, the output pixel becomes fully transparent.
6. The default minimum opaque pixel ratio is `0.1`.
7. For tiles that are not considered transparent, RGB median must be calculated using opaque pixels.
8. For tiles that are not considered transparent, alpha median must also be calculated using opaque pixels.
9. If a tile has no opaque pixels, the output pixel must be `(0, 0, 0, 0)`.

These threshold values must be separated into variables so they can be adjusted later.

Create a 32×32 canvas or image data.

Apply each tile’s representative color to the corresponding one of the 1024 output pixels.

Generate the result as PNG.

Because an actual 32×32 image is too small to preview clearly, show an enlarged preview on screen.

Use CSS to preserve the pixel art look:


# 8. Algorithm Implementation Notes

The core of this program is the 32×32 conversion algorithm.

The algorithm must work as follows:

1. Draw the original image onto a temporary canvas at original size.
2. Get the pixel data of the original image.
3. Divide the original image into 32 columns and 32 rows.
4. Collect pixels for each tile.
5. For each tile, calculate median values for R, G, B, and A channels.
6. Put the calculated median value into the corresponding output pixel.
7. Create a final 32×32 canvas.
8. Use the result canvas for preview and download.

Images whose dimensions are not evenly divisible by 32 must still be handled.

Examples:

* A 100×100 image must still be processed into 32×32.
* A 1920×1080 image must still be processed into 32×32.
* A 17×17 image, smaller than 32×32, must still be handled as stably as possible.
